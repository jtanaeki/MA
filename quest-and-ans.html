What is the difference between ngOnInit and constructor?

ngOnInit is a lifecycle hook that runs after the constructor and is used for “actual code work”. The constructor is the default method of a class that is used primarily to initialize class properties and dependency injection

Name and Describe some Angular Lifecycle Hooks

ngOnChanges(): called when Angular sets data-bound input properties for a component. For example, when you’re looking to watch data changes in observables.

ngOnInit(): called when Angular initializes the component AFTER the input properties have been set

ngDoCheck(): called when a change occurs that Angular doesn’t detect

ngDestroy(): called right before Angular removes a component. For example, when unsubscribing and cleaning up data when using observables

What is NgRx?

It's a framework, similar to Redux, that’s used for state management and to build reactive applications in angular. It involves a store, effects, reducers, and actions.

Can you tell me about React hooks?

React hooks are used by functional components in order to access state and lifecycle methods, whereas component hooks would not need hooks. Some examples of React hooks include useState and useEffect.
